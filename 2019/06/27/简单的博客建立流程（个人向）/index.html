简单的博客建立流程（个人向）

头痛，作为一个学计算机的怎么能连个博客都没有。。

这篇文档主要用于自己进行回顾复习如何从0开始建立的博客

第一步(准备阶段)：

1.你需要有一个邮箱，例如xxx@gmail.com，最好不要用qq邮箱；

2.注册Github帐号；

3.完成以上两个步骤就可以在主机上准备环境了。

第二步(适配环境)：

1.准备 node 和 git 环境：

  • 安装NodeJS

  • 安装Git

  • 验证是否安装成功：

    安装成功后打开git bash（Windowns）或者终端（Mac），下方中将统一称为命令行。
    在命令行中输入相应命令验证是否成功，如果成功会有相应的版本号。

2.安装 Hexo:

  • 在自己喜欢的位置建立一个新的文件夹用来存放与博客相关的东西，我设置的路径是在桌面C:\Users\DELL\Desktop\myblog；
  • 在该目录下右键点击Git Bash Here，打开git的控制台窗口，当然也可以使用Powershell来输入指令（仅限安装操作）；
  • 输入npm install hexo-cli -g安装Hexo 会有几个报错，无视即可；
  • 然后输入npm install hexo --save继续安装；
  • 安装完后输入hexo -v验证是否安装成功；
  • 然后就要初始化网站，输入hexo init初始化文件夹，接着输入npm install安装必备的组件；
  • 输入hexo g生成静态网页，然后输入hexo s打开本地服务器，然后浏览器打开http://localhost:4000/，就可以看到我们的博客了，效果如下：
  • 在Bash窗口里按 ctrl + c即可关闭本地服务器。

第三步(连接Github与本地)：

1.首先进入github官网登陆自己的帐号，在右上角新建自己的仓库（New repository)

2.然后如下图所示，输入自己的项目名字，后面一定要加.github.io后缀，README初始化也要勾上。

3.然后项目就建成了，点击Settings，向下拉到最后有个GitHub Pages，点击Choose a theme选择一个主题。然后等一会儿，再回到GitHub Pages，会变成下面这样：

用户名和邮箱根据你注册github的信息自行修改;

然后生成密钥SSH key：

输入eval "$(ssh-agent -s)"，添加密钥到ssh-agent。

再输入ssh-add ~/.ssh/id_rsa，添加生成的SSH key到ssh-agent。

打开github，在头像下面点击settings，再点击SSH and GPG keys，新建一个SSH，名字随便。

打开C:\Users\Administrator\.ssh\id_rsa.pub，注意是隐藏文件夹，将其中的内容复制到新建的SSH中。

输入ssh -T git@github.com，如果如下图所示，出现你的用户名，那就成功了。

由于我在看教程安装的过程中遇到了种种原因，最后我没有用Github进行绑定，而在室友帮助下用的腾讯云，总之就是repository修改为你自己的github项目地址就好了（之前有个.io的网址）。

第四步(发布文章)：

1.首先在博客根目录下右键打开git，安装一个扩展npm install hexo-deployer-git --save。

然后输入hexo new post "article title"，新建一篇文章；

2.然后打开C:\Users\DELL\Desktop\myblog\source\_posts的目录，可以发现下面多了一个文件夹和一个.md文件，一个用来存放你的图片等数据，另一个就是你的文章文件；

3.编写完markdown文件后，根目录下输入hexo g生成静态网页，然后输入hexo s可以本地预览效果，最后输入hexo d上传到github上。这时打开你的github.io主页就能看到发布的文章啦。

第五步(绑定域名)：

是不是觉得自己的xxx.github.io很辣鸡，甚至没有让人一下就想点进去的欲望呢？

没有关系，学会这波买域名，你就能解锁.cn .com .xyz等一系列操作。

→_→我就这一步搞得晕头转向的（可能还是有点混乱）；

1.在腾讯云开发者平台注册自己的帐号，同时在腾讯云上也注册自己的帐号（两个是不一样的东西）;

2.新建项目，在左边小列表里找到静态pages服务，进入设置项目地址；

3.个人账户->SSH公钥中添加自己静态网址的公钥；

4.在腾讯云中个人认证，我购买的是.cn域名16元/年，在解析中设置路径

5.进入域名管理填写自己的个人信息和资料等待审核；

6.在审核结束之后进入腾讯云开发者平台的Pages服务中绑定自己的新域名，比如我自己的Chrisbear.cn，顺便开启强制HTTPS访问；

7.验证测试看输入自己买的域名能不能正常登陆，如果跟我一样是404的话说明没有进行步骤6；

拓展（自己的服务器以及更快的静态博客打开方式）：

1.注册Cloudflare帐号，绑定自己的域名

2.将他给的DNS号复制到云解析中的DNS号中修改

3.设置一系列你需要的东西；

自己的服务器还没有买，这个以后更新，暂时博客的事情我也才弄懂到这儿。
